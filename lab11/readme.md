# Лабораторная работа 9

- Тема: **Nullability**.

## Вопросы

- Что такое `null`?

  null — это специальное значение, означающее «ничего тут нет». Это отсутствие объекта или данных.

- Почему тип `int` не позволяет значение `null`?
  
  Потому что int это тип-значение. Все value types всегда имеют конкретное значение в памяти (0, 5, -3). Они НЕ могут быть «пустыми».

- Что означает `string?`?
  
  Это строка, которая может быть null

- Почему `int?` и `string?` работают по-разному? Как вообще работает `int?`?
  
string — reference type, и наличие ? просто говорит компилятору: «Разрешаю быть null».

int — value type, поэтому int? превращается в Nullable<int> — специальную структуру с двумя полями:
- Value: значение типа int
- HasValue: есть ли значение

То есть int? — это уже полноценный объект-обёртка.

- Что делает оператор `!`? 
  
  Это null-forgiving operator — оператор «я гарантирую, что здесь точно НЕ null».
```
string? s = null;
Console.WriteLine(s!);
```

Ты говоришь компилятору: «я знаю, что делаю».
Но если переменная реально null → будет ошибка во время выполнения.

- Приведите пример, как с помощью оператора `!` можно обойти гарантии nullability компилятора.  

```
string? name = GetName();
Console.WriteLine(name!.Length);
```

Компилятор НЕ жалуется, но если name == null → программа рухнет.

```
Person? p = null;
Console.WriteLine(p!.ToString());
```

Компилятор думает, что всё норм → на деле — авария.

- Чего позволяет достичь `required`?

 required заставляет программиста обязать заполнить свойство перед созданием объекта.
